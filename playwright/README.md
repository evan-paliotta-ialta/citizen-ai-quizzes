# Course Builder — Playwright Scripts

Automates the creation of the Citizen AI Developer Program in Microsoft 365:
SharePoint pages for all 16 modules, Microsoft Forms quizzes, Excel progress tracker.

---

## Before You Start

**1. Fill in your tenant name in `config.js`**

Open `config.js` and set `tenant` to the part before `.sharepoint.com` in your SharePoint URL.

```js
tenant: 'ialta',   // ← change this
```

**2. Confirm Node.js is installed**
```bash
node --version   # should be v16 or higher
```

**3. Install dependencies** (already done if you're reading this)
```bash
npm install
```

---

## Run Order

Run these scripts in order. Each is safe to re-run — already-created items are skipped.

### Step 1 — Save your Microsoft 365 session (run once)
```bash
node auth/save-auth.js
```
A browser window opens. Log in to your Microsoft 365 account including MFA.
The session is saved to `auth/auth.json`. Re-run whenever your session expires (typically every 24–72 hours).

---

### Step 2 — Create the SharePoint site
```bash
node scripts/01-create-site.js
```
Creates a Communication Site at `https://[tenant].sharepoint.com/sites/CitizenAI`.
If the site already exists, this step is skipped automatically.

---

### Step 3 — Create all module pages
```bash
node scripts/02-create-pages.js
```
Creates:
- Home/landing page with course overview and navigation
- 16 module lesson pages
- 16 quiz pages
- Final Exam page

Reads content from `../course-content/*.md` files.
Runs headlessly. Progress is logged to the console.

---

### Step 4 — Create Microsoft Forms
```bash
node scripts/03-create-forms.js
```
Creates:
- 16 module quiz Forms (one per module)
- Final Exam Form
- Capstone Submission Form

Runs with visible browser (`headless: false`) for reliability with the Forms UI.
Form URLs are saved to `forms-urls.json` — do not delete this file.

---

### Step 5 — Set up the progress tracker
```bash
node scripts/04-setup-tracker.js
```
Creates `Course_Progress_Tracker.csv` in the SharePoint Documents library.
Creates an Administrator Reference page with all form links and the license gate workflow.

---

### Step 6 — Verify everything
```bash
node scripts/05-verify.js
```
Checks every page, form URL, and file. Prints a pass/fail checklist.
For any failures, re-run the corresponding script and then re-run verify.

## Troubleshooting

**"No auth session found"**
Run `node auth/save-auth.js` and log in again. Sessions expire after 24–72 hours.

**Page creation fails with 403**
Your account may not have Site Owner permissions on the SharePoint site.
Ask your SharePoint admin to add you as a Site Owner for the CitizenAI site.

**Forms script fails mid-way**
Re-run `node scripts/03-create-forms.js`. Already-created forms are saved in `forms-urls.json`
and will be skipped. The script picks up where it left off.

**SharePoint page content looks wrong**
The Markdown-to-HTML converter in `utils/content-loader.js` handles common patterns.
If a specific page has formatting issues, you can edit it manually in SharePoint after creation.

**Verify shows failures after re-running a script**
Wait 60 seconds and run verify again — SharePoint page indexing can have a short delay.
