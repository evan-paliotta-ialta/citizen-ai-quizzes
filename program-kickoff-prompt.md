# Citizen AI Engineer Program - Kickoff Prompt

**Date:** 2026-03-02

## Context

There is a new program I have been tasked with figuring out/leading where my company will be giving out Claude licenses to people who ask for them. We want the non-technical folks to use Claude and Claude Code but we have to be ISO 27001 and SOC2 compliant.

## Analogy: The Driver's License

We're using the analogy of a driver's license. We'll be making everyone who gets the license take a course so they can know the rules of the road and the processes for following the laws/rules of the road. We will be making everyone sign an agreement and developing processes (our internal laws) that they must follow.

## Two Primary Action Items

1. **Developing the laws and best practices** - The policies, guardrails, and compliance framework
2. **Designing a course to teach said laws and best practices** - Training/certification program

## Levers We Can Pull

- **Global skills files** added to everyone's Claude Code instance during setup
- **Structured `agent.md` and `claude.md` files** to set the guardrails
- **Including the signed agreement** on all projects
- **GitHub-first workflow** - Everything in GitHub, nothing stored locally
- **Periodic log summarization** for audit trail of what citizen AI engineers are doing
- **Jira ticket process** - Every task gets a Jira ticket for audit trail, then point Claude to the ticket to complete it
- **Tiered process rigor** - Basic querying may not need full process, while other tasks require more rigorous workflow

## Key Compliance Requirements

- ISO 27001 compliant
- SOC2 compliant
- Full audit trail
- Agreement/acknowledgment from all users

## Research Needed

- How other companies are rolling out AI tools in SOC2/ISO 27001 compliant manner
- Best practices for non-technical AI tool governance
- Patterns for AI usage policies and agreements
